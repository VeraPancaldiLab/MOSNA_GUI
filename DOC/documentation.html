Mosna Analysis GUI Documentation (exemple)

Ce document est une documentation fictive destinée à illustrer l’intégration d’une aide utilisateur dans l’interface. Le contenu peut être remplacé par la documentation réelle du projet.

1. Objectif de l’interface

Mosna Analysis GUI est une interface graphique conçue pour préparer une configuration YAML et lancer plusieurs scripts d’analyse de façon guidée. L’utilisateur choisit un dossier de travail, ajuste des paramètres par onglets, sauvegarde le fichier de configuration, puis exécute les scripts.

2. Démarrage et dossier de travail

Au lancement, un choix de dossier est obligatoire. Sans working directory, l’interface est désactivée.

  • Working directory : emplacement où les scripts écrivent leurs sorties et lisent des données.
  • Si l’utilisateur annule le choix au démarrage, l’application se ferme.
Astuce : crée un dossier par analyse (par exemple un dossier par patient ou par batch) pour garder des sorties propres.

3. Onglets de configuration

Les paramètres sont organisés en onglets. Chaque onglet correspond à une section du fichier YAML. Les champs s’affichent automatiquement selon leur type (texte, booléen, liste, etc.).

Onglet Contenu Exemples de paramètres
Documentation Page d’aide intégrée (HTML) affichée dans l’interface. Procédure, explications, conventions de fichiers, FAQ.
Tysserand Paramètres liés à la construction et aux features réseau. Nodes directory, CPU, Edges method, Min neighbors, Extension, colonnes X/Y, phenotype.
Assortativity Paramètres pour les métriques d’assortativité et analyses associées. Network directory, Phenotype column, Index.
NAS Paramètres pour l’analyse de niches (clustering, réduction dimensionnelle, statistiques). Niches method, Processing method, clusterer_type, reducer_type, metric, n_neighbors, resolution.
Attention : si certains champs attendent une convention de nommage (patient / sample), assure-toi que les fichiers en entrée respectent cette convention.

4. Sauvegarde de la configuration

Le bouton Save Config écrit les paramètres dans le fichier YAML principal. L’interface applique une validation légère des types (entier, float, booléen, liste, etc.).

  • Les champs vides ou “None” sont enregistrés comme null dans le YAML.
  • Les listes peuvent être saisies avec une syntaxe Python simple, par exemple [1, 2, 3].
  • Les champs texte multi-lignes sont affichés avec une zone dédiée.

5. Lancement des scripts

La barre de boutons Run … lance les scripts définis dans l’application. Les scripts sont exécutés dans un thread séparé pour ne pas bloquer l’interface.

Scripts disponibles

  • TYSSERAND NETWORK : construction des réseaux et calcul de features.
  • ASSORTATIVITY : calcul des métriques d’assortativité.
  • NICHE ANALYSIS : pipeline NAS / SCAN-IT selon la configuration.
  • CLEAR TEMPORARY FILES : nettoyage des fichiers temporaires.

Progression et messages

Les scripts peuvent renvoyer des messages “compatibles GUI” sur la sortie standard, par exemple : 

[QT_INFO] Chargement des données
[QT_PROGRESS] current=15 total=100 desc=Calcul des features réseau
[QT_PROGRESS] current=100 total=100 desc=Terminé

Ces lignes mettent à jour la barre de progression et le statut dans l’interface.

6. Dépannage rapide (exemples)

Le script échoue avec un message long

L’interface affiche un extrait de la sortie dans une boîte de dialogue. En cas de besoin, relance le script en ligne de commande dans le même environnement conda pour obtenir le log complet.

Les onglets ne sont pas accessibles au lancement

Cela arrive si aucun dossier de travail n’a été défini. Choisis un dossier valide.

Problèmes de dépendances

Si un import échoue, vérifie que l’environnement utilisé est bien celui de la GUI. Exemple : 

conda activate mosna-GUI
python -c "import numpy; import pandas; print('OK')"
